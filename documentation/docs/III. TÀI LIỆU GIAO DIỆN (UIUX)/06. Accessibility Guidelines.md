## **2. WCAG Compliance**

### **2.1. Compliance Level**

- **Target**: WCAG 2.1 Level AA
- **Minimum**: WCAG 2.1 Level A

### **2.2. Core Principles**

1. **Perceivable**: Thông tin phải có thể nhận biết được
2. **Operable**: Giao diện phải có thể vận hành được
3. **Understandable**: Thông tin và UI phải dễ hiểu
4. **Robust**: Nội dung phải đủ mạnh mẽ để các công nghệ hỗ trợ có thể diễn giải

## **3. Color & Contrast**

### **3.1. Color Contrast Ratios**

| Element Type | Ratio | Standard |
|--------------|-------|----------|
| **Normal Text** | 4.5:1 | WCAG AA |
| **Large Text** (18px+) | 3:1 | WCAG AA |
| **UI Components** | 3:1 | WCAG AA |
| **Graphical Objects** | 3:1 | WCAG AA |

### **3.2. Color Combinations**

**Pass:**
- Primary Blue (#2563EB) on White: 4.6:1 ✓
- Gray 900 (#111827) on White: 16.8:1 ✓
- Error Red (#DC2626) on White: 5.1:1 ✓

**Fail:**
- Primary Blue Light (#3B82F6) on White: 2.9:1 ✗
- Gray 500 (#6B7280) on White: 3.2:1 ✗ (borderline)

### **3.3. Don't Rely on Color Alone**

- **Error States**: Use icon + color + text
- **Required Fields**: Use asterisk + label + color
- **Links**: Use underline + color
- **Status Indicators**: Use icon + text + color

**Example:**
```jsx
// ❌ Bad
<div style={{ color: 'red' }}>Error</div>

// ✅ Good
<div>
  <Icon name="error" />
  <span style={{ color: 'red' }}>Error: Invalid input</span>
</div>
```

## **4. Typography**

### **4.1. Font Sizes**

- **Minimum**: 14px (body text)
- **Recommended**: 16px (body text)
- **Large Text**: 18px+ (for better readability)

### **4.2. Line Height**

- **Minimum**: 1.4
- **Recommended**: 1.5-1.6
- **Large Text**: 1.3

### **4.3. Font Weight**

- **Body Text**: 400 (Regular)
- **Headings**: 600-700 (SemiBold/Bold)
- **Avoid**: Text lighter than 400

### **4.4. Text Spacing**

- **Letter Spacing**: Minimum -0.5px
- **Word Spacing**: Normal
- **Paragraph Spacing**: Minimum 1.5x font size

## **5. Keyboard Navigation**

### **5.1. Tab Order**

- Logical order (top to bottom, left to right)
- All interactive elements focusable
- Skip links for main content

### **5.2. Focus Indicators**

**Requirements:**
- Visible focus outline
- Minimum 2px width
- Contrast ratio 3:1
- Color: Primary Blue (#2563EB)

**Example:**
```css
:focus {
  outline: 2px solid #2563EB;
  outline-offset: 2px;
}
```

### **5.3. Keyboard Shortcuts**

| Action | Shortcut |
|--------|----------|
| Submit Form | Enter |
| Cancel/Close | Escape |
| Navigate Menu | Arrow Keys |
| Select Item | Enter/Space |

### **5.4. Focus Management**

- **Modals**: Trap focus inside modal
- **Dynamic Content**: Move focus to new content
- **Form Errors**: Focus first error field
- **Page Load**: Focus main content (skip header)

## **6. Screen Readers**

### **6.1. Semantic HTML**

Use proper HTML elements:

```html
<!-- ✅ Good -->
<button>Submit</button>
<a href="/login">Login</a>
<input type="email" />

<!-- ❌ Bad -->
<div onClick={handleClick}>Submit</div>
<span onClick={navigate}>Login</span>
```

### **6.2. ARIA Labels**

**When to Use:**
- Icon-only buttons
- Decorative images
- Complex widgets
- Form fields without visible labels

**Examples:**

```jsx
// Icon button
<button aria-label="Close modal">
  <Icon name="close" />
</button>

// Form field
<input 
  type="email" 
  aria-label="Email address"
  aria-required="true"
/>

// Status message
<div role="alert" aria-live="polite">
  Transaction completed successfully
</div>
```

### **6.3. ARIA Roles**

| Role | Usage |
|------|-------|
| `button` | Clickable element that's not a button |
| `dialog` | Modal dialogs |
| `alert` | Error/success messages |
| `navigation` | Navigation regions |
| `main` | Main content area |
| `banner` | Site header |
| `complementary` | Sidebar content |

### **6.4. ARIA States**

```jsx
// Disabled state
<button aria-disabled="true">Submit</button>

// Expanded state
<button aria-expanded="false" aria-controls="menu">
  Menu
</button>

// Required field
<input aria-required="true" />

// Invalid field
<input aria-invalid="true" aria-describedby="error-id" />
```

## **7. Forms**

### **7.1. Labels**

**Requirements:**
- Every input must have a label
- Labels associated with inputs (for/id)
- Required fields clearly marked

**Example:**
```jsx
<label htmlFor="email">
  Email <span aria-label="required">*</span>
</label>
<input 
  id="email" 
  type="email" 
  required 
  aria-required="true"
/>
```

### **7.2. Error Messages**

**Requirements:**
- Clear, descriptive error messages
- Associated with input (aria-describedby)
- Announced by screen readers (aria-live)

**Example:**
```jsx
<label htmlFor="email">Email</label>
<input 
  id="email"
  type="email"
  aria-invalid={hasError}
  aria-describedby="email-error"
/>
{hasError && (
  <div id="email-error" role="alert">
    Please enter a valid email address
  </div>
)}
```

### **7.3. Field Instructions**

- Provide instructions before user interacts
- Use aria-describedby for complex fields
- Show examples when helpful

## **8. Images**

### **8.1. Alt Text**

**Requirements:**
- All images must have alt text
- Decorative images: empty alt=""
- Informative images: descriptive alt text
- Functional images: describe function

**Examples:**

```jsx
// Informative
<img src="chart.png" alt="Transaction volume increased 25% this month" />

// Decorative
<img src="decoration.png" alt="" />

// Functional
<img src="search-icon.png" alt="Search" />
```

### **8.2. Image Guidelines**

- Alt text should be concise (< 125 chars)
- Don't start with "Image of..." or "Picture of..."
- Describe content, not appearance
- Include text in images as alt text

## **9. Interactive Elements**

### **9.1. Buttons**

**Requirements:**
- Descriptive button text
- Minimum size: 44x44px
- Clear focus state
- Loading state announced

**Example:**
```jsx
<button 
  aria-label={loading ? "Submitting..." : "Submit form"}
  aria-busy={loading}
>
  {loading ? "Submitting..." : "Submit"}
</button>
```

### **9.2. Links**

**Requirements:**
- Descriptive link text (avoid "Click here")
- Visually distinct (underline + color)
- Focus state visible
- External links announced

**Example:**
```jsx
// ❌ Bad
<a href="/login">Click here</a>

// ✅ Good
<a href="/login">Login to your account</a>
```

### **9.3. Modals**

**Requirements:**
- Focus trap inside modal
- ESC key closes modal
- Focus returns to trigger on close
- aria-labelledby for title
- aria-describedby for content

**Example:**
```jsx
<Modal
  aria-labelledby="modal-title"
  aria-describedby="modal-description"
  onClose={handleClose}
>
  <h2 id="modal-title">Confirm Action</h2>
  <p id="modal-description">Are you sure you want to proceed?</p>
</Modal>
```

## **10. Content Structure**

### **10.1. Headings**

**Requirements:**
- Proper heading hierarchy (h1 → h2 → h3)
- One h1 per page
- Don't skip levels
- Descriptive headings

**Example:**
```html
<h1>Dashboard</h1>
  <h2>Account Balance</h2>
  <h2>Recent Transactions</h2>
    <h3>This Week</h3>
    <h3>This Month</h3>
```

### **10.2. Landmarks**

Use semantic HTML5 elements:

```html
<header role="banner">...</header>
<nav role="navigation">...</nav>
<main role="main">...</main>
<aside role="complementary">...</aside>
<footer role="contentinfo">...</footer>
```

### **10.3. Lists**

Use proper list elements:

```html
<!-- ✅ Good -->
<ul>
  <li>Item 1</li>
  <li>Item 2</li>
</ul>

<!-- ❌ Bad -->
<div>Item 1</div>
<div>Item 2</div>
```

## **11. Dynamic Content**

### **11.1. Live Regions**

**Requirements:**
- Use aria-live for dynamic updates
- polite: Wait for pause
- assertive: Interrupt immediately

**Example:**
```jsx
<div aria-live="polite" aria-atomic="true">
  {notification}
</div>
```

### **11.2. Loading States**

**Requirements:**
- Announce loading state
- Announce completion
- Provide progress if possible

**Example:**
```jsx
<div aria-live="polite" aria-busy={loading}>
  {loading ? "Loading transactions..." : "Transactions loaded"}
</div>
```

## **13. Common Issues & Solutions**

### **13.1. Missing Alt Text**

**Issue:** Images without alt text  
**Solution:** Add descriptive alt text or empty alt for decorative images

### **13.2. Low Contrast**

**Issue:** Text doesn't meet contrast ratio  
**Solution:** Use darker text or lighter background

### **13.3. Missing Focus Indicators**

**Issue:** Can't see focus state  
**Solution:** Add visible focus outline

### **13.4. Form Labels Missing**

**Issue:** Inputs without labels  
**Solution:** Add labels or aria-labels

### **13.5. Keyboard Traps**

**Issue:** Can't navigate away from element  
**Solution:** Ensure all interactive elements are keyboard accessible
