## **2. Breakpoints**

### **2.1. Standard Breakpoints**

| Breakpoint | Min Width | Device Type | Container Width |
|------------|-----------|------------|-----------------|
| **Mobile** | 0px | Smartphone | 100% (padding: 16px) |
| **Tablet** | 768px | Tablet | 100% (padding: 24px) |
| **Laptop** | 1024px | Laptop | 1024px (padding: 32px) |
| **Desktop** | 1280px | Desktop | 1280px (padding: 32px) |
| **Large Desktop** | 1536px | Large Desktop | 1536px (padding: 32px) |

### **2.2. Media Queries**

```css
/* Mobile First Approach */
/* Base styles for mobile */

/* Tablet */
@media (min-width: 768px) {
  /* Tablet styles */
}

/* Laptop */
@media (min-width: 1024px) {
  /* Laptop styles */
}

/* Desktop */
@media (min-width: 1280px) {
  /* Desktop styles */
}
```

## **3. Layout Strategies**

### **3.1. Mobile Layout (< 768px)**

**Principles:**
- Single column layout
- Full-width components
- Stack vertically
- Touch-friendly targets (min 44x44px)
- Bottom navigation (optional)

**Examples:**

**Dashboard:**
```
┌─────────────────┐
│  Navigation     │
├─────────────────┤
│  User Info Card │
├─────────────────┤
│  Balance Card   │
├─────────────────┤
│  [Deposit]      │
├─────────────────┤
│  [Withdraw]     │
├─────────────────┤
│  [Transfer]     │
├─────────────────┤
│  [History]      │
└─────────────────┘
```

**Forms:**
- Full-width inputs
- Stack labels above inputs
- Full-width buttons
- Larger touch targets

**Tables:**
- Convert to card layout
- Each row becomes a card
- Hide less important columns
- Swipe actions (optional)

### **3.2. Tablet Layout (768px - 1024px)**

**Principles:**
- 2-column layout where appropriate
- Sidebar navigation (collapsible)
- Table view with horizontal scroll
- Medium-sized components

**Examples:**

**Dashboard:**
```
┌──────────┬──────────┐
│ User Info│ Balance  │
├──────────┴──────────┤
│  [Deposit] [Withdraw]│
│  [Transfer] [History]│
├──────────────────────┤
│  Recent Transactions  │
└──────────────────────┘
```

**Forms:**
- 2-column layout for related fields
- Medium-sized inputs
- Side-by-side buttons

### **3.3. Desktop Layout (> 1024px)**

**Principles:**
- Multi-column layout
- Full sidebar navigation
- Full table view
- Hover states
- More whitespace

**Examples:**

**Dashboard:**
```
┌──────┬──────────┬──────────┐
│ Side │ User Info│ Balance  │
│ bar  ├──────────┴──────────┤
│      │  [Deposit] [Withdraw]│
│      │  [Transfer] [History] │
│      ├──────────────────────┤
│      │  Recent Transactions  │
└──────┴──────────────────────┘
```

**Forms:**
- 2-3 column layout
- Larger inputs
- More spacing

## **4. Component Responsive Behavior**

### **4.1. Navigation**

**Mobile:**
- Hamburger menu (top-left)
- Drawer/sidebar khi mở
- Bottom navigation (optional)

**Tablet:**
- Collapsible sidebar
- Icon + text labels

**Desktop:**
- Full sidebar
- Always visible
- Icon + text labels

### **4.2. Cards**

**Mobile:**
- Full width
- Stack vertically
- Padding: 16px

**Tablet:**
- 2-column grid
- Padding: 20px

**Desktop:**
- 3-4 column grid
- Padding: 24px

### **4.3. Tables**

**Mobile:**
- Card layout
- Each row = one card
- Key info only
- Swipe for actions

**Tablet:**
- Table view
- Horizontal scroll
- All columns visible

**Desktop:**
- Full table
- No scroll needed
- All columns visible
- Hover states

### **4.4. Forms**

**Mobile:**
- Single column
- Full-width inputs
- Labels above inputs
- Full-width buttons

**Tablet:**
- 2-column for related fields
- Labels inline or above
- Side-by-side buttons

**Desktop:**
- 2-3 column layout
- Labels inline (left-aligned)
- Button groups

### **4.5. Modals**

**Mobile:**
- Full-screen
- Bottom sheet style
- Swipe to dismiss

**Tablet:**
- Centered modal
- Max width: 600px
- Backdrop click to close

**Desktop:**
- Centered modal
- Max width: 800px
- Backdrop click to close

## **7. Images & Media**

### **7.1. Image Responsive**

```css
img {
  max-width: 100%;
  height: auto;
}
```

### **7.2. Aspect Ratios**

- Maintain aspect ratios
- Use `object-fit: cover` for backgrounds
- Lazy loading for images below fold

### **8.1. Minimum Sizes**

- **Mobile**: 44x44px (iOS/Android standard)
- **Tablet**: 48x48px
- **Desktop**: 40x40px (hover states)

### **8.2. Spacing Between Targets**

- **Mobile**: Minimum 8px
- **Tablet**: Minimum 12px
- **Desktop**: Minimum 16px

## **9. Performance Considerations**

### **9.1. Mobile Optimization**

- Lazy load images
- Code splitting
- Minimize JavaScript
- Optimize fonts (subset, preload)
- Use CSS transforms for animations

### **9.2. Network Considerations**

- Show loading states
- Optimistic updates
- Cache API responses
- Offline support (optional)

## **11. Common Patterns**

### **11.1. Mobile-First Card Grid**

```css
.grid {
  display: grid;
  grid-template-columns: 1fr;
  gap: 16px;
}

@media (min-width: 768px) {
  .grid {
    grid-template-columns: repeat(2, 1fr);
    gap: 24px;
  }
}

@media (min-width: 1024px) {
  .grid {
    grid-template-columns: repeat(3, 1fr);
    gap: 32px;
  }
}
```

### **11.2. Responsive Table**

```css
.table {
  display: block;
  overflow-x: auto;
}

@media (min-width: 768px) {
  .table {
    display: table;
  }
}
```

### **11.3. Responsive Navigation**

```css
.nav {
  display: none; /* Mobile: hidden */
}

.nav-mobile {
  display: block; /* Mobile: hamburger */
}

@media (min-width: 1024px) {
  .nav {
    display: block; /* Desktop: visible */
  }
  
  .nav-mobile {
    display: none; /* Desktop: hidden */
  }
}
```

## **12. Accessibility in Responsive Design**

### **12.1. Touch Accessibility**

- Adequate touch targets
- No hover-only interactions
- Clear visual feedback

### **12.2. Screen Reader**

- Proper heading hierarchy
- ARIA labels
- Skip links
- Focus management

### **12.3. Keyboard Navigation**

- All interactive elements accessible
- Focus indicators visible
- Logical tab order
